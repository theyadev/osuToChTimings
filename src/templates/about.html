About .osu to .chart Timing Converter

This is a tool designed to help rhythm game charters convert timing points from osu! beatmaps to Clone Hero chart format. This web application makes it easy to extract BPM changes and time signatures from any osu! beatmap and generate a complete .chart file ready for use in Moonscraper.

Features

  • Direct download of osu! beatmaps from the official site
  • Automatic extraction of timing data from .osu files
  • Precise conversion of osu! timing points to .chart's tick-based format
  • Supports multiple BPM changes and time signatures
  • Generates complete .chart files with metadata
  • Provides audio file download from the beatmap

How It Works

  1. Downloading: The application downloads the beatmap from the osu! website.
  2. Extraction: It extracts the .osu file and audio from the downloaded beatmap package.
  3. Parsing: The application parses the timing points and metadata from the .osu file.
  4. Conversion: osu! timing points (in milliseconds) are converted to chart's tick-based system using a formula that ensures precise alignment.
  5. Output: The application generates a complete .chart file with SyncTrack section and proper metadata.

Technical Details

osu! stores timing information in milliseconds, while .chart uses a tick-based system (192 ticks per beat by default). The conversion process calculates the exact tick positions based on elapsed time and BPM changes to ensure accurate timing.

The formula used for conversion is: ticks = elapsed_minutes * BPM * tick_rate

For negative timing points in osu! files (which can occur in some beatmaps), the converter handles them by moving them to the earliest equivalent position in the .chart file, as .chart does not support negative ticks.

Common Issues

  • Bad beatmap quality: If the original osu! beatmap has poor timing quality, these inaccuracies will be carried over to the .chart file; choose well-timed beatmaps (ranked, qualified, loved) for best results.
  • Audio misalignment: If the timing feels off in your editor, ensure you're using the identical audio file from the osu! beatmap.
  • Server limitations: Very large beatmaps might time out or fail to process due to serverless function limits.
  • Negative timing points: The tool handles negative timing points by moving them to the closest equivalent position, which might not be 100% accurate.
